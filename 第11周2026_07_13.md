### Day 1 — 07.17（I2C 协议攻克日）

> 时间预算：4-5 小时高强度

#### 上午/下午任务（180分钟）：跟学 I2C + AT24C02

看江协科技第 20-21 集。

**Part 1：I2C 总线底层时序（90分钟）**

你必须能独立写出的四个基本函数：

```c
// 1. 起始条件
void I2C_Start() {
    SDA = 1;        // 先拉高 SDA
    SCL = 1;        // 再拉高 SCL
    Delay5us();     // 延时（满足时序要求）
    SDA = 0;        // SCL 高电平期间 SDA 拉低 → 起始条件
    Delay5us();
    SCL = 0;        // 拉低 SCL，准备发送数据
}

// 2. 停止条件
void I2C_Stop() {
    SDA = 0;        // 先拉低 SDA
    SCL = 1;        // 拉高 SCL
    Delay5us();
    SDA = 1;        // SCL 高电平期间 SDA 拉高 → 停止条件
    Delay5us();
}

// 3. 发送一个字节并等待应答
unsigned char I2C_SendByte(unsigned char dat) {
    unsigned char i;
    for(i = 0; i < 8; i++) {
        SDA = (dat >> 7);  // 先发最高位
        dat <<= 1;
        SCL = 1;           // 拉高时钟，从设备读取数据
        Delay5us();
        SCL = 0;
    }
    // 等待应答
    SDA = 1;               // 释放 SDA（上拉电阻拉高）
    SCL = 1;
    Delay5us();
    if(SDA) {              // 如果 SDA 还是高，说明从设备没有应答
        SCL = 0;
        return 0;          // 失败
    }
    SCL = 0;
    return 1;              // 成功
}

// 4. 接收一个字节并发送应答
unsigned char I2C_RecvByte() {
    unsigned char i, dat = 0;
    SDA = 1;               // 释放 SDA
    for(i = 0; i < 8; i++) {
        dat <<= 1;
        SCL = 1;
        Delay5us();
        if(SDA) dat |= 1;  // 读取 SDA 状态
        SCL = 0;
    }
    return dat;
}
```

**你必须要理解的坑**：

| 坑 | 说明 |
|----|------|
| 为什么需要 Delay5us()？ | I2C 标准速率 100kHz，每个脉冲约 10μs，延时是为了满足时序要求 |
| 为什么 SDA 用 `sbit SDA = P2^0` 但实际可能接在其他引脚？ | 51 单片机没有硬件 I2C，全用 IO 口模拟（"软件 I2C"或"IO 模拟 I2C"） |
| 开漏输出 + 上拉电阻 | IO 口只能输出低电平或高阻态，高电平靠上拉电阻拉上去 |

**Part 2：AT24C02 读写（90分钟）**

AT24C02 的器件地址：0xA0（写）/ 0xA1（读）

写一个字节到 AT24C02：

```c
void AT24C02_WriteByte(unsigned char addr, unsigned char dat) {
    I2C_Start();
    I2C_SendByte(0xA0);      // 器件地址+写
    I2C_SendByte(addr);      // 存储地址
    I2C_SendByte(dat);       // 数据
    I2C_Stop();
    Delay10ms();             // 等待内部写入完成（AT24C02 需要最多 5ms）
}
```

读一个字节：

```c
unsigned char AT24C02_ReadByte(unsigned char addr) {
    unsigned char dat;
    I2C_Start();
    I2C_SendByte(0xA0);      // 先写地址
    I2C_SendByte(addr);
    I2C_Start();             // 重新开始
    I2C_SendByte(0xA1);      // 器件地址+读
    dat = I2C_RecvByte();
    I2C_Stop();
    return dat;
}
```

验证：写一个字节到地址 0x00，读出来看是否一致。如果读出来是 0xFF 或全 0，大概率是时序不对或者应答失败。

### Day 2 — 07.18（PWM 与蜂鸣器 + 传感器入门）

> 时间预算：4-5 小时高强度

#### 上午/下午任务（180分钟）：PWM 与蜂鸣器

看江协科技第 22 集（蜂鸣器与 PWM）。

**Part 1：PWM 原理（60分钟）**

你必须要理解：

- PWM（脉冲宽度调制）= 控制方波的占空比（高电平时间占总周期的比例）
- 占空比 50% → 电机转一半速度、LED 半亮、蜂鸣器音量中等
- 51 单片机没有硬件 PWM 输出引脚，用定时器中断 + 软件翻转 IO 口模拟

呼吸灯实现：

```c
unsigned char pwm_duty = 0;   // 占空比（0-100）
unsigned char pwm_cnt = 0;    // 当前计数

void Timer0_ISR() interrupt 1 {
    TH0 = ...;  // 100μs 定时
    TL0 = ...;

    pwm_cnt++;
    if(pwm_cnt >= 100) pwm_cnt = 0;  // 100 级 PWM 周期

    if(pwm_cnt < pwm_duty) LED = 0;  // 亮
    else LED = 1;                     // 灭
}

// 主循环中逐渐改变占空比
```

**Part 2：蜂鸣器播放音乐（120分钟）**

你已有 LCD1602，结合蜂鸣器做一个"音乐盒"：

- LCD1602 第一行显示当前播放的音符
- 按键矩阵选曲（至少 2 首曲子：生日快乐 + 两只老虎）
- 蜂鸣器播放，按键可暂停/继续

#### 今晚任务（120分钟）：DS18B20 温度传感器（可选，优先级低于 PWM）

如果你 PWM 和蜂鸣器已经调通，开始看江协科技第 23 集（DS18B20）。

DS18B20 是单总线（OneWire）协议——只有一根数据线，时序要求比 I2C 更严格（微秒级）。核心难点：延时精度。51 单片机用 12MHz 晶振时，一个机器周期 1μs，需要精确控制微秒级延时。

如果你今晚还有精力：读出 DS18B20 的温度值，显示在 LCD1602 上。

## 五、已知难点预警

### 三日计划 #1 的坑

| 坑 | 说明 | 对策 |
|----|------|------|
| 波特率不准 | 晶振 11.0592MHz vs 12MHz，后者波特率有误差 | 用 11.0592MHz（开发板默认） |
| 串口中断与 Timer1 冲突 | Timer1 被串口占用，不能再做普通定时 | Day3 单独练习 Timer1 的定时功能（临时切换模式） |
| 字符串接收边界 | 没有 `\0` 概念，需手动管理缓冲区 | 用固定长度或 `\r\n` 结束符 |
| RI/TI 混淆 | RI=接收完成，TI=发送完成，共用一个中断号 | 中断函数内用 `if (RI)` / `if (TI)` 区分 |

### 三日计划 #2 的坑

| 坑 | 说明 | 对策 |
|----|------|------|
| I2C 时序不满足 | Delay5us 不够精确，导致 SCL/SDA 跳变时序错乱 | 用逻辑分析仪或示波器确认波形，没有就用软件延时反复调 |
| AT24C02 写周期 | 写完一个字节后立即读会失败 | 写完必须 Delay10ms 等待内部写入 |
| 上拉电阻缺失 | 开发板可能没焊上拉电阻 | 确认 P2.0/P2.1 有 4.7kΩ 上拉到 VCC，没有就自己焊或用内部弱上拉 |
| PWM 精度不够 | 100 级 PWM 在低频时肉眼可见闪烁 | 提高 PWM 频率（减小定时器周期）或增加级数 |
| DS18B20 时序苛刻 | 微秒级延时，12MHz 晶振下 `_nop_()` 约 1μs | 用示波器或逻辑分析仪确认，没有就反复校准延时函数 |
| 多模块整合时中断冲突 | UART 中断 + Timer0 中断 + 外部中断可能互相干扰 | 设置合理的中断优先级，中断服务函数尽量短 |
