**任务1（60分钟）**：手写Dijkstra最短路径算法。

**核心概念**（动手前用15分钟理解）：
- Dijkstra解决的是：从图中一个起点出发，到达所有其他顶点的最短距离。
- 核心思想：贪心+松弛。每次从“未确定最短距离的顶点”中选出当前距离最小的那个，然后更新它所有邻居的距离。
- 数据结构选择：用std::priority_queue（小根堆）存储（距离, 顶点），每次取堆顶就是当前距离最小的顶点。这和你上周的堆排是同一个技术栈。
- 不能处理负权边——这是Dijkstra的局限（有负权边需要用Bellman-Ford）。

**要求**：
1. 用邻接表存储图：`vector<vector<pair<int, int>>> graph`，每个pair是（邻居顶点, 边权）。
2. 实现 `vector<int> dijkstra(int start, int n, vector<vector<pair<int, int>>>& graph)`，返回从start到所有顶点的最短距离数组。
3. 使用 `std::priority_queue<pair<int, int>, vector<pair<int, int>>, greater<>>` 作为小根堆。
4. **验证**：创建5个节点的图：
   - 0-1: 4, 0-2: 1
   - 1-3: 1
   - 2-1: 2, 2-3: 5
   - 3-4: 3
   从0出发，最短距离应为：0→0: 0, 0→1: 3, 0→2: 1, 0→3: 4, 0→4: 7。
5. **思考**（写注释）：为什么Dijkstra用priority_queue而不用普通queue？如果所有边权都是1，Dijkstra退化成什么？（答案：退化成BFS——你上周学的）

**纪律**：这是你第一次把图、堆、贪心三种知识组合起来解决一个实际问题。Dijkstra是面试图题最高频考点（之一）。如果卡住，先画图手动模拟一遍算法过程。

3. 核心代码（手动计算版，不到15行）：

```python
import math

def cosine_sim(a, b):
    dot = sum(x*y for x,y in zip(a,b))
    norm_a = math.sqrt(sum(x*x for x in a))
    norm_b = math.sqrt(sum(x*x for x in b))
    return dot / (norm_a * norm_b)

# 模拟词向量（2维简化版）
cat = [0.8, 0.6]
dog = [0.7, 0.7]
car = [-0.5, 0.3]

print(f"猫和狗的相似度: {cosine_sim(cat, dog):.3f}")    # 应该较高
print(f"猫和汽车的相似度: {cosine_sim(cat, car):.3f}")  # 应该较低
```

4. 运行这段代码，把输出结果记录到`AI-Notes/Embedding实操_0601.md`。

### Day 2：最小生成树（Prim/Kruskal）+ 暑假项目技术选型深化（周二，6月2日）

#### 【主线：数据结构+工程预研 | 90分钟 | 18:30-20:00】

**任务1（60分钟）**：手写最小生成树算法（二选一，建议先Prim）。

**核心概念**（动手前用10分钟理解）：
- 最小生成树（MST）：在一个带权无向连通图中，找到一棵包含所有顶点的树，且所有边的权值之和最小。
- Prim算法：从任意一个顶点开始，每次选择"连接已选顶点集合和未选顶点集合的权重最小的边"。这和Dijkstra几乎一模一样，只是“距离”的含义不同——Dijkstra累加路径长度，Prim只看单条边的权重。
- Kruskal算法：把所有权重从小到大排序，依次选边，如果这条边不会形成环就加入。需要用到并查集（Disjoint Set Union）判断是否成环。

**要求**：
1. 实现 `int prim(vector<vector<pair<int, int>>>& graph, int n)`，返回MST的总权重。
2. 使用和Day1相同的priority_queue结构（只是比较逻辑略有不同）。
3. **验证**：创建和Day1相同的5个节点的图，但边是无向的：
   - 0-1: 4, 0-2: 1
   - 1-2: 2, 1-3: 1
   - 2-3: 5
   - 3-4: 3
   Prim算法从0出发，MST总权重应为 1+2+1+3 = 7。
4. **思考**（写注释）：为什么你昨天学的Dijkstra和今天学的Prim可以用同一套代码框架（都用了priority_queue做贪心选择）？它们本质区别在哪里？

**可选扩展**（有余力才做）：如果你对并查集感兴趣，尝试实现Kruskal算法。并查集的两个核心操作是find（找根节点）和union（合并两个集合），实现很简单但思想很精妙。

**今日警戒线**：Prim算法正确输出MST总权重。
