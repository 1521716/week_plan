### Day 1：快速排序日 + AI排序学习概念（周一，5月25日）

#### 【主线：数据结构+算法 | 90分钟 | 18:30-20:00】

**任务**：手写快速排序，理解分区（partition）是快排的灵魂。

**核心概念（动手前用15分钟理解）**：
- 快排本质：选一个pivot（基准），把数组分成"比pivot小的"和"比pivot大的"两部分，再递归处理左右两部分。
- 快排的精髓全在partition函数：它用O(n)时间把数组切成两块，返回pivot的最终位置。
- 最坏情况：每次选到最小/最大元素作为pivot，退化为O(n²)。随机化pivot或三数取中可以避免。
- 为什么叫"快速"排序：实际运行速度通常比归并和堆排快——缓存友好，常数因子小。

**要求**：

- 实现经典快排：`void quickSort(vector<int>& arr, int left, int right)`。
- 实现partition函数：`int partition(vector<int>& arr, int left, int right)`，选最右元素为pivot。
- 在partition内部：用两个指针i和j，i指向"小于pivot的区域的最后一个位置"，j遍历数组，遇到比pivot小的就交换到i+1位置。这是Lomuto分区方案，面试最常用。
- 验证：对 `{3,1,4,1,5,9,2,6,5,3,5}` 排序，输出 `{1,1,2,3,3,4,5,5,5,6,9}`。
- 思考（写注释）：为什么partition函数用引用传参而不是返回值？为什么快排不稳定？（相同元素的相对顺序可能改变）

**纪律**：全程不使用AI生成代码。快排是排序中最经典的手写题，面试考查频率极高。如果partition逻辑卡住了，用纸上画图模拟一遍——数组[5,2,8,3,7,4]，pivot=4，手动走一遍分区过程。

### Day 2：归并排序日 + AI模型评估深入（周二，5月26日）

#### 【主线：数据结构+算法 | 90分钟 | 18:30-20:00】

**任务**：手写归并排序（递归版 + 迭代版），理解分治思想的两种实现方式。

**核心概念（动手前用10分钟理解）**：
- 归并排序本质：递归地把数组对半切，切到只剩一个元素（天然有序），再两两合并有序数组。
- 核心操作是merge：把两个已经排好序的子数组合并成一个有序数组。
- 归并排序是稳定的（相同元素的相对顺序不变），这是它对比快排的重要优势。
- 缺点：需要O(n)额外空间。

**要求**：

- 递归版：`void mergeSort(vector<int>& arr, int left, int right)` + `void merge(vector<int>& arr, int left, int mid, int right)`。merge函数用两个临时数组或一个辅助数组。
- 迭代版（底部向上）：`void mergeSortIterative(vector<int>& arr)`。不用递归，用一个外层循环控制子数组大小（size从1开始翻倍），每次把相邻两个大小为size的有序子数组合并。
- 验证：用Day1相同的数组 `{3,1,4,1,5,9,2,6,5,3,5}` 测试两种实现，确保结果一致。
- 思考（写注释）：为什么说归并排序是"外部排序"的基础？（外部排序=数据太大内存装不下，需要用到磁盘——归并的思想天然适合分批读写）

**纪律**：迭代版归并排序比较难写，注意边界条件（mid和right的计算）。如果卡住超过30分钟，先完成递归版，迭代版留到周五缓冲日。

### Day 3：堆排序日 + 三种O(n log n)排序横向对比（周三，5月27日）

#### 【主线：数据结构+算法 | 90分钟 | 18:30-20:00】

**任务**：手写堆排序，然后完成三种排序算法的横向对比。

**核心概念（动手前用10分钟理解）**：
- 堆排序利用了你上周实现的堆数据结构。两个步骤：
- 步骤1（建堆）：把无序数组变成一个最大堆——从最后一个非叶子节点开始，逐个sink，O(n)时间。
- 步骤2（排序）：反复把堆顶（最大值）和堆末尾交换，堆大小减1，然后sink新的堆顶。O(n log n)时间。
- 堆排序是不稳定的，空间复杂度O(1)（原地排序）——这是它对比归并排序的最大优势。
- 最后一个非叶子节点的索引是 (n/2) - 1。

**要求**：

- 用你上周的堆知识，实现堆排序 `void heapSort(vector<int>& arr)`。
- 不要用std::priority_queue，手写heapify（建堆）和排序过程。
- 辅助函数：`void heapify(vector<int>& arr, int n, int i)`——对以i为根的子树进行sink操作，n是当前堆的大小。
- 验证：用Day1相同的数组测试。

**横向对比总结（写在一个代码注释块里，这是今天的核心产出）**：

```text
========== 三种O(n log n)排序横向对比 ==========

| 算法     | 平均时间 | 最坏时间 | 最好时间 | 空间复杂度 | 稳定性 | 适用场景                 |
|----------|----------|----------|----------|------------|--------|--------------------------|
| 快速排序 | O(nlogn) | O(n²)    | O(nlogn) | O(logn)    | 不稳定 | 通用排序，实际中最快     |
| 归并排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(n)       | 稳定   | 需要稳定排序/外部排序    |
| 堆排序   | O(nlogn) | O(nlogn) | O(nlogn) | O(1)       | 不稳定 | 内存极受限/只需top K     |

关键理解：
- 快排为什么实际最快？缓存友好，常数因子小，且可以原地排序。
- 归并排序为什么稳定？merge时遇到相等元素，优先取左半部分的。
- 堆排序为什么实际比快排慢？跳跃访问内存（父子节点在数组中位置不连续），缓存不友好。
```

### 重要提示

如果快排的分区过程让你卡住，记住：partition是快排的灵魂，也是最容易写错的部分。i和j的初始位置、交换时机、pivot的位置——这些细节在实际面试中80%的人都会写错。你只需要在纸上手动走一遍例子，然后记住一个原则：**i始终指向"小于pivot的区域的最后一个元素"，j负责往前探索**。
