# 📅 详细单周冲锋计划（2026.05.05 - 2026.05.11）

## 第1天：5月5日（周二）

**今日目标**：彻底搞定C++的类与对象基础，并确立你的AI使用原则。

### · 18:30 - 20:00 (90分钟) 主线：C++ 类与对象

- **任务**：手写一个封装良好的 Student 类。
- **具体要求**：
  1. 属性：学号（std::string）、姓名（std::string）、语文/数学/英语成绩（int）。
  2. 方法：
     - 一个带参数的构造函数，用于初始化所有成员。
     - 一个无参数的构造函数，将所有数值成员初始化为0。
     - 一个计算平均分的成员函数 double average()。
     - 一个析构函数，用 std::cout 打印 "Student [姓名] destroyed."。
  3. 验证：在 main 函数中使用 new 创建一个 Student 对象，调用 average() 函数并打印结果，最后用 delete 销毁对象，观察析构函数的输出。
- **纪律**：全程不许问 AI，只能查书和资料。这是为了强迫自己回忆和查阅语法。

### · 20:10 - 21:10 (60分钟) 辅线：Prompt 核心与命名实体

#### 任务1 (30分钟)：体验 Prompt 质量差异。

1. 打开你的AI工具。
2. 先用一个极差的 Prompt 问它："写个学生管理系统"。看看它给你生成什么垃圾。
3. 再用下面这个标准的好 Prompt 问它：
   ```prompt
   角色：你是一位资深的C++后端面试官。
   任务：请帮我生成一个"学生成绩管理系统"中，最核心的"Student"类的设计草案。
   要求：
   1. 只设计 Student 类，不要写 main 函数。
   2. 属性必须包含学号、姓名、三门课的成绩。
   3. 方法必须包含构造函数、析构函数、计算平均分的函数。
   4. 用代码块展示，并加上简要注释。
   5. 最后，用一段话解释你为什么要这样设计这个类。
   ```
4. 严格对比：把两次得到的回复截图保存到 AI-Notes 文件夹，图片命名为 Prompt_对比_0505.png。思考：为什么第二个回复好得多？

#### 任务2 (30分钟)：学习 AI 名词。

1. 问 AI："用最通俗的例子解释什么是 Token（词元）？" 和 "什么是大模型幻觉（Hallucination）？举一个程序员写代码时会遇到的例子。"
2. 动手做笔记：在 AI-Notes 文件夹新建一个 AI名词解释_0505.md 文件，用你自己的一句话总结这两个词，然后贴上你觉得最形象的例子。

## 第2天：5月6日（周三）

**今日目标**：体验C++标准模板库(STL)的威力，了解AI的计费与上下文模型。

### · 18:30 - 20:00 (90分钟) 主线：STL 初体验

- **任务**：使用 std::stack 和 std::queue 容器，不自己造轮子。
- **具体要求**：
  1. 栈操作：引入 <stack>，创建一个能存 int 的栈。循环 push 数字 1-5，再用循环 pop 并打印。自己在注释里解释 push、pop、top、empty 各自干了什么。
  2. 队列操作：引入 <queue>，创建 int 队列。push 1-5，再循环 pop 打印。
  3. 思考与注释：在你的代码最后，用块注释回答："对比之前的C语言链表实现，使用 std::stack 的好处是什么？代价或缺点可能是什么？"
- **纪律**：同样，全程先自己手写，跑通。有问题，先用 Ctrl+J 在网上搜 std::stack 的用法。

### · 20:10 - 21:10 (60分钟) 辅线：Token 经济学与上下文窗口

#### 任务1 (20分钟)：概念深化。

问 AI："什么是大模型的上下文窗口（Context Window）？为什么我跟他聊久了，他会忘记之前说的话？" 并问它："Gemini、GPT、Claude 的上下文窗口一般有多长，大概能装下几本《三体》？"

#### 任务2 (40分钟)：亲手试试。

1. 去找一个比较大一点的 C++ 源文件，或者用你之前写的几百行代码。
2. 在 AI 对话框里点"上传文件"，让它总结这个代码的功能。
3. 问它："我的上下文窗口还剩多少？这个文件占了多少 Token？"
4. 感受：这是你第一次把"Token"这个抽象概念和你自己的代码文件大小联系起来。把你的感受记到 AI名词解释_0506.md 里。

## 第3天：5月7日（周四）

**今日目标**：挑战C++中最容易出错的拷贝控制（深拷贝），并学会用AI审查自己的代码。

### · 18:30 - 20:00 (90分钟) 主线：深拷贝与浅拷贝

- **任务**：手写一个动态管理内存的 MyString 类。
- **具体要求**：
  1. 这个类有一个 char* 的私有成员，用于存储字符串。
  2. 实现：带参构造函数（接收 const char*）、拷贝构造函数、拷贝赋值运算符、析构函数（释放内存）。
  3. 验证：在 main 里创建两个 MyString 对象，用一个去拷贝构造另一个，然后修改其中一个，观察另一个是否被影响。如果被影响，说明你写的是浅拷贝，这是 bug，要改。
- **纪律**：你大概率会写出内存泄漏或崩溃的代码，这是必经之路。先自己调试，改不出来可以先放一放。

### · 20:10 - 21:10 (60分钟) 辅线：AI 作为代码审查员

#### 任务1 (30分钟)：用 AI 帮你找 bug。

1. 把你刚写的、可能有 bug 的 MyString 类代码发给 AI。
2. 使用下面的标准 Prompt：
   ```prompt
   角色：你是一位C++语言律师和内存安全专家。
   任务：审查我下面的MyString类代码。
   要求：
   1. 只关注深拷贝/浅拷贝相关的内存泄漏、重复释放问题。
   2. 用通俗的语言解释问题出在哪，为什么这是 bug。
   3. 严禁直接给我能运行的完整代码。只指出问题位置和修复思路。
   ```
3. 根据它的提示，你自己去改代码，直到 main 函数里的验证逻辑正确。

#### 任务2 (30分钟)：学习 AI 名词。

1. 问 AI："什么是 RAG（检索增强生成）？用大白话和生活中的例子解释。"
2. 更新你的 AI名词解释_0505.md 文件，加上 RAG 的解释和例子。

## 第5天：5月9日（周六）

**今日目标**：用C++重写成绩管理系统的核心模块，并学习"AI嵌入"的概念。

### · 09:00 - 11:30 (150分钟) 主线：C++ 文件操作实战

- **任务**：用 C++ 的 fstream 重写你之前成绩管理系统的文件读写模块。
- **具体要求**：
  1. 复习《C++ Primer》或网上的 fstream 章节。
  2. 写两个函数：
     - saveToFile(const vector<Student>& students, const string& filename)：将学生数据以你定义的格式（如CSV或JSON）存入文件。
     - loadFromFile(vector<Student>& students, const string& filename)：从文件中读取数据，解析并填充到一个 vector<Student> 里。
  3. 注意：这里我们第一次用到了 vector 容器。你可能还不熟，没关心，查资料，先用起来。它的用法比链表简单太多。
- **验证**：先 save 几个学生，清空内存，再 load 它们并打印，确保数据完全一致。

### · 13:00 - 14:30 (90分钟) 辅线：初探 Embedding（嵌入）

#### 任务1 (45分钟)：理解概念。

问 AI："什么是 Embedding（向量嵌入）？为什么它能衡量两段文字有多相关？请用比喻和最简单的例子解释。"

#### 任务2 (45分钟)：动手实验。

让 AI 帮你写一个不超过 20 行的 Python 脚本，把你成绩管理系统里几个不同学生的信息（姓名、成绩）转换成向量，并算出两两之间的"相似度"。你的目标不是成为 Python 大神，而是亲眼看到"相似度数字"的跳动，理解底层逻辑。
