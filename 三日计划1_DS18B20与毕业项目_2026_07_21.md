#### Part 1：单总线底层时序（90分钟）

你必须要能独立写出的三个基本函数：

```c
// DS18B20初始化
bit DS18B20_Init() {
    bit ack;
    DQ = 0;              // 主机拉低DQ
    Delay480us();        // 至少480μs
    DQ = 1;              // 释放DQ
    Delay60us();         // 等待15-60μs
    ack = DQ;            // 读取DS18B20应答（0=存在，1=不存在）
    Delay420us();        // 等待时序结束
    return ack;
}

// 写一个字节
void DS18B20_WriteByte(unsigned char dat) {
    unsigned char i;
    for(i = 0; i < 8; i++) {
        DQ = 0;           // 拉低总线，启动写时序
        _nop_();          // 延时1μs
        DQ = dat & 0x01;  // 发送最低位
        Delay60us();      // 保持60μs
        DQ = 1;           // 释放总线
        dat >>= 1;        // 准备下一位
    }
}

// 读一个字节
unsigned char DS18B20_ReadByte() {
    unsigned char i, dat = 0;
    for(i = 0; i < 8; i++) {
        DQ = 0;           // 拉低总线，启动读时序
        _nop_();          // 延时1μs
        DQ = 1;           // 释放总线
        _nop_();          // 延时1μs
        dat >>= 1;        // 先右移，准备接收新位
        if(DQ) dat |= 0x80; // 如果DQ为1，最高位置1
        Delay60us();      // 保持60μs
    }
    return dat;
}
```

**你必须理解的坑**：

| 坑 | 说明 |
|----|------|
| 为什么时序要求这么严格？ | 单总线没有时钟线，完全靠延时计时。延时函数必须精确到微秒级，用_nop_()或空循环。如果晶振是12MHz，一个机器周期1μs，一个_nop_()约1-2μs |
| DS18B20的两种供电模式 | 寄生供电（用DQ线的上拉供电）和外部供电。你的开发板大概率是外部供电（三根线：VCC/GND/DQ），接线前先确认 |
| 读取温度总是返回85°C | 这是DS18B20上电默认值，说明没读到真实温度，检查初始化是否成功 |

#### Part 2：温度转换与读取（90分钟）

DS18B20的标准操作流程：

```c
// 启动温度转换
DS18B20_Init();
DS18B20_WriteByte(0xCC);  // 跳过ROM匹配（只有一个设备）
DS18B20_WriteByte(0x44);  // 启动温度转换
Delay750ms();              // 等待转换完成（12位精度需要最多750ms）

// 读取温度
DS18B20_Init();
DS18B20_WriteByte(0xCC);  // 跳过ROM匹配
DS18B20_WriteByte(0xBE);  // 读取暂存器
temp_low = DS18B20_ReadByte();   // 低8位
temp_high = DS18B20_ReadByte();  // 高8位

// 换算成实际温度
int temp = (temp_high << 8) | temp_low;  // 合并成16位
float temperature = temp * 0.0625;        // 12位分辨率
```
