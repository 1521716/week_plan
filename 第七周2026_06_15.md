## 短期作战计划：江协科技51单片机入门（2026.06.18 - 2026.06.19）

**零、为什么先学51单片机**

你已经选了硬件方向，选了STM32平台，但现在直接从STM32起步会有三个障碍：STM32的HAL库封装太厚，你看不到底层；CubeMX自动生成大量代码，你分不清哪些是核心逻辑；时钟树和外设配置复杂，容易从入门到放弃。

51单片机恰好是解决这些问题的跳板——没有CubeMX、没有HAL库，所有配置都要你亲手写寄存器。芯片结构简单（只有4个IO口、2个定时器、1个串口），两周就能学完核心外设。学完51再回头用STM32，你会清楚HAL库到底帮你封装了什么。

江协科技这套教程评价高，原因就是"从零开始、手把手、每行代码都解释"。

这两天你的任务就是：**搭建51开发环境，跟学前5-6集视频，亲手写出第一个点亮LED的程序，理解"软件到底怎么控制硬件"**。

#### Day 2：独立按键控制 + 第一个小综合项目（周五，6月19日）

【上午：独立按键检测 | 09:00-12:00 | 180分钟】

**任务1（90分钟）：跟学按键检测，理解输入模式**。

跟学视频：第5集（独立按键控制LED）

**要求**：
1. 理解按键的电路原理：为什么需要上拉电阻？按键按下时引脚读到的是高电平还是低电平？
2. 跟着视频写代码，核心逻辑：
   ```c
   sbit KEY = P3^0;   // 假设按键接在P3.0
   sbit LED = P1^0;
   
   void main() {
       while(1) {
           if(KEY == 0) {      // 按键按下（假设低电平有效）
               LED = 0;        // 点亮LED
           } else {
               LED = 1;        // 熄灭LED
           }
       }
   }
   ```
3. **你一定会遇到的问题**：按键按住时LED一直亮，松开就灭——这是"按下亮"逻辑。但如果需求是"按一下切换状态"呢？视频中会讲到按键消抖和状态检测。
4. **按键消抖原理**（这是今天第一个核心概念）：
   - 机械按键按下时，电平会抖动（在0和1之间快速跳变），持续约5-20ms
   - 软件消抖：检测到按键按下后，延时20ms再次检测，如果还是按下状态才确认为有效按下
   - 这个延时和你昨天写的Delay函数是同一个

**验证**：编译通过。有实物则烧录验证：按下按键LED亮，松开LED灭。

**任务2（90分钟）：改造练习——实现按键状态切换**。

不看视频，自己实现以下功能（这是对你理解程度的真实检验）：

**练习1（按键状态切换）**：按一下按键，LED亮；再按一下，LED灭；再按一下，LED亮……（Toggle功能）

提示思路（自己先想10分钟，想不出来再看）：
- 你需要检测的不是"按键是否按下"，而是"按键是否从松开变为按下"——这个瞬间称为"下降沿"
- 用一个变量记录上一次的按键状态，和当前状态比较，就能检测到这个瞬间

**练习2（按键计数器）**：每按一次按键，LED闪烁对应次数（按1次闪烁1下，按2次闪烁2下……最多9次，第10次归零）。闪烁期间按键不响应。

**纪律**：如果卡住了，先自己想20分钟→问AI思路（不要代码）→关掉AI自己写。写不出来没关系，这个练习本来就是用来暴露问题的。
